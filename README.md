### 브랜치 네이밍 규칙
브랜치 종류/이슈넘버-기능이름

e.g) feature/#5-login

### 브랜치 전략
gitflow 전략을 따르지만, release 브랜치가 없는 형태 

main : 최종 배포 브랜치

develop : 개발 단계 브랜치 (디폴트)

### 커밋 메세지 컨벤션

| 타입       | 설명                      |
|----------|-------------------------|
| feat     | 새로운 기능을 추가              |
| fix      | 버그를 수정                  |
| docs     | 문서와 관련된 변경 사항을 기록       |
| style    | 코드 포맷팅                  |
| refactor | 리팩토링 작업을 기록             |
| test     | 테스트 코드를 추가하거나 수정        |
| chore    | 초기세팅 및 코드에 영향을 주지 않는 작업 |
